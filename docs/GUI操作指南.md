## 二、添加节点（守护进程连接）

**如果节点丢了，最快恢复：**
```bash
docker compose restart mcsm-web
```
重启后节点配置自动加载，不需要手动添加。

**如果重启没用，再手动添加：**
左侧菜单 → **「节点管理」** → **「添加节点」**

| 字段 | 值 |
|------|-----|
| 名称 | `主服务器` |
| IP 地址 | `catzizimiku.top` |
| 端口 | `24444` |
| 密钥 | `credentials.md` 中的「守护进程密钥」 |

点 **「确认」**，节点状态变绿即为连接成功。

## 三、创建 Minecraft 服务器实例

**操作路径：**
左侧菜单 → **「终端」** → 点击 **「主服务器」** 节点 → 点右上角 **「创建实例」**

### 第一步：填写基本信息

必填：
- **类型：** `Minecraft Java 版`
- **版本：** `Forge 1.20.1`
- **实例名称：** 随意，如 `Forge 1.20.1`

选填（全部留空）：
- **宿主机工作目录：** 留空（默认创建在 `/home/yuan/minecraft-server/mcsm/daemon/data/InstanceData/<实例UUID>/`）
- **启动命令：** 留空（镜像自带默认入口）
- **关闭实例命令：** 留空（默认 `^C`，Docker 模式会自动 stop）
- **安装服务端命令：** 留空（镜像自动下载 Forge）
- **配置更新命令：** 留空
- **基础端口：** 留空（自动分配）
- **文件编码：** 留空（默认 UTF-8）
- **Ping 配置：** 留空（默认检测 25565 端口）
- **终端选项：** 留空（默认即可）
- **事件任务（自动启动/自动重启）：** 留空，按需勾选

点 **「确认创建」**，等待几秒实例创建完成。

### 第二步：配置 Docker 模式

实例创建后默认是通用模式，需要改为 Docker 模式才能用 `itzg/minecraft-server` 镜像。

**操作路径：**
点进实例 → **「设置」** → 找到 **「Docker 配置」**

开启 **「Docker 模式」**，然后填写以下内容：

#### 基础设置

| 字段 | 值 |
|------|-----|
| 镜像名 | `itzg/minecraft-server:java17` |
| 容器名 | `mc-forge`（随意填） |
| 工作目录 | `/data` |
| 挂载实例目录到工作目录 | ✅ 开启 |
| 内存限制 | `0`（不限制）或 `4294967296`（4G） |
| 网络模式 | `bridge` |

#### 端口映射

点「添加端口映射」，加三条：

| 宿主机端口 | 容器端口 | 协议 |
|-----------|---------|------|
| `25565` | `25565` | `tcp` |
| `25575` | `25575` | `tcp` |
| `24454` | `24454` | `udp` |

#### 环境变量

点「添加环境变量」，加八条：

| 变量名 | 值 |
|-------|-----|
| `EULA` | `TRUE` |
| `TYPE` | `FORGE` |
| `VERSION` | `1.20.1` |
| `FORGE_VERSION` | `47.4.21` |
| `ONLINE_MODE` | `false` |
| `MEMORY` | `4G` |
| `MAX_PLAYERS` | `20` |
| `TZ` | `Asia/Shanghai` |

#### 额外挂载卷

点「添加挂载卷」：

| 宿主机路径 | 容器路径 | 模式 |
|-----------|---------|------|
| `/home/yuan/minecraft-server/mods` | `/mods` | `ro` |

保存设置。

### 安装整合包

在「环境变量」里加一条 `CF_SERVER_MOD`，镜像会自动下载整合包：

| 变量名 | 值 |
|-------|-----|
| `CF_SERVER_MOD` | `https://www.curseforge.com/minecraft/modpacks/<整合包名>` |

首次启动时自动下载安装，等待即可。

## 四、上传 Mod

**操作路径：**
实例 → **「文件管理」** → 进入 `mods/` 文件夹（没有就新建）

**上传方法：**
- 点 **「上传文件」**，从本机选择 `.jar` 文件
- 或者拖拽文件到窗口
- 或者用 **「远程下载」** 输入 Mod 直链

**Mod 源文件位置：** `~/minecraft-server/mods/` 目录下，可以直接从这里上传。

> ⚠️ 创建新实例时不要默认挂载 `mods/` 目录，除非用户明确要求。

---

## 五、上传世界存档

**操作路径：**
实例 → **「文件管理」** → 实例根目录

- 上传 `world/` 文件夹（整个存档）
- 上传 `server.properties`（覆盖配置）
- 上传 `eula.txt`（确保 `eula=true`）

---

## 六、启动实例

**操作路径：**
实例 → **「控制台」** → 点 **「开启实例」**

首次启动约 3-5 分钟，看到以下日志即为成功：
```
Done (XX.XXXs)! For help, type "help"
```

### 常用命令

| 命令 | 说明 |
|------|------|
| `/op 玩家名` | 给管理员 |
| `/gamemode creative 玩家名` | 创造模式 |
| `/give 玩家名 物品ID` | 给物品 |
| `/say 消息` | 公告 |

## 八、节点离线排查

**1. 重启面板（最快）：**
```bash
docker compose restart mcsm-web
```

**2. 检查防火墙：**
终端运行 `! firewall-config`，确保 `FedoraWorkstation` 和 `FedoraServer` 两个区域都勾选了 `mc-server` 服务。

**3. 手动添加节点：**
见上面「二、添加节点」。

---

## 九、命令行备份

也有命令行脚本可以用（配合 crontab 定时备份更方便）：

```bash
# 交互式备份（菜单选实例）
./scripts/backup.sh

# 列出可备份的实例
./scripts/backup.sh --list

# 备份指定实例
./scripts/backup.sh forge-1.20.1
```
