# MCSManager GUI 操作指南

> 面板地址：`http://catzizimiku.top:23333`
> ⚠️ 用 **HTTP**，不是 HTTPS

---

## 一、登录

1. 浏览器打开 `http://catzizimiku.top:23333`
2. 输入账号密码（见 `credentials.md`）
3. 点「登录」

---

## 二、添加节点（守护进程连接）

> 如果节点被删除或首次配置需要手动连接。

**操作路径：**
左侧菜单 → **「节点管理」** → **「添加节点」**

**填写表单：**

| 字段 | 值 |
|------|-----|
| 名称 | `主服务器` |
| IP 地址 | `catzizimiku.top` |
| 端口 | `24444` |
| 密钥 | 见 `credentials.md` 中的「守护进程密钥」 |

点 **「确认」**，节点状态变绿即为连接成功。

---

## 三、创建 Minecraft 服务器实例

**前置条件：** 节点已添加且在线（绿色状态）。

**操作路径：**
左侧菜单 → **「终端」** → 点击 **「主服务器」** 节点

### 步骤

1. 在节点页面，点右上角 **「创建实例」**
2. 弹出窗口中选择：
   - **类型：** `Minecraft Java 版`
   - **版本：** `Forge 1.20.1`
   - **实例名称：** 随意填写，如 `Forge 1.20.1`
3. 点 **「确认创建」**
4. 等待几秒，实例创建完成

---

## 四、上传 Mod

**操作路径：**
左侧菜单 → **「终端」** → **「主服务器」** → 点进你的实例 → **「文件管理」**

### 上传 Mod 文件

1. 在文件列表中进入 `mods/` 目录（如果没有，点「新建文件夹」创建）
2. 点右上角 **「上传文件」**（或拖拽文件到窗口）
3. 从本机选择 `.jar` 文件上传
4. 或者用 **「远程下载」**：输入 Mod 的直链下载地址

### Mod 源文件位置

服务器上已有的 Mod 在 `~/minecraft-server/mods/` 目录，可以直接使用。

---

## 五、上传世界存档

**操作路径：**
实例 → **「文件管理」** → 实例根目录

1. 点 **「上传文件」** → 选择 `world/` 文件夹上传
2. 或者用 **「远程下载」** 从备份地址下载
3. 也可以上传 `server.properties`、`eula.txt` 等配置文件

---

## 六、启动实例

**操作路径：**
左侧菜单 → **「终端」** → **「主服务器」** → 点进你的实例 → **「控制台」**

1. 点 **「开启实例」** 按钮
2. 等待控制台输出日志
3. 首次启动需要下载 Forge 并加载 Mod，约 3-5 分钟
4. 看到 `Done (XX.XXXs)! For help, type "help"` 即为启动成功

### 常用控制台命令

| 命令 | 说明 |
|------|------|
| `/op 玩家名` | 给予管理员权限 |
| `/gamemode creative 玩家名` | 切换创造模式 |
| `/give 玩家名 物品ID` | 给物品 |
| `/say 消息` | 公告消息 |

---

## 七、日常管理

| 操作 | 路径 |
|------|------|
| 启动/停止 | 实例 → 控制台 → 开启/关闭 |
| 上传/下载文件 | 实例 → 文件管理 |
| 查看日志 | 实例 → 控制台 / 日志 |
| 修改配置 | 实例 → 文件管理 → 编辑 `server.properties` |
| 定时备份 | 实例 → 定时任务 |

---

## 八、如果看不到节点

**检查防火墙：**
打开 `firewall-config`（终端运行 `! firewall-config`）
- **FedoraWorkstation** 区域 → 勾选 `mc-server` 服务
- **FedoraServer** 区域 → 勾选 `mc-server` 服务

**重启面板：**
```bash
docker compose restart mcsm-web
```

**重新添加节点：**
按上面「二、添加节点」的步骤重新添加。
