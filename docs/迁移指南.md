# 服务器迁移指南

> 目标：换一台机器，能从 git 恢复整个服务器。

---

## 需要备份的文件

以下文件不在 git 中，迁移前需要手动备份：

| 文件 | 位置 | 说明 |
|------|------|------|
| Mod 文件 | `mods/*.jar` | 从原服务器复制或重新下载 |
| 世界存档 | `mcsm/daemon/data/InstanceData/<uuid>/world/` | 如果不想丢失进度 |
| DDNS-GO 配置 | `ddns-go-data/.ddns_go_config.yaml` | 含阿里云 API 密钥 |
| 凭据 | `credentials.md` | 所有密码和密钥 |

---

## 迁移步骤

### 新机器上

```bash
# 1. 克隆项目
git clone git@github.com:userzbb/mc-server-1.20.1-forge.git
cd mc-server-1.20.1-forge

# 2. 复制 Mod 文件
cp /path/to/backup/mods/*.jar mods/

# 3. 恢复凭据
cp /path/to/backup/credentials.md .

# 4. 启动 MCSManager
docker compose up -d

# 5. 恢复世界存档（如果备份了）
cp -r /path/to/backup/world mcsm/daemon/data/InstanceData/<uuid>/world

# 6. 在面板中创建实例
#    - 打开 http://服务器IP:23333
#    - 终端 → 主服务器 → 创建实例
#    - 选 Minecraft Java 版 / Forge 1.20.1
#    - 创建后用 instance-config.json 的内容配置 Docker 设置
#    - 复制 server.properties 到实例的文件管理中
```

### 旧机器上备份

```bash
# Mod
cp mods/*.jar /backup/

# 世界存档
cp mcsm/daemon/data/InstanceData/<uuid>/world /backup/

# DDNS 配置
cp ddns-go-data/.ddns_go_config.yaml /backup/

# 凭据
cp credentials.md /backup/
```
