# 服务器迁移指南

> 目标：换一台机器，能从 git 恢复整个服务器。

---

## 需要备份的文件

以下文件不在 git 中，迁移前需要手动备份：

| 文件 | 位置 | 说明 | 重要性 |
|------|------|------|--------|
| Mod 文件 | `mods/*.jar` | 从原服务器复制或重新下载 | ✅ 必选 |
| **世界存档（核心）** | `mcsm/daemon/data/InstanceData/<uuid>/world/` | 包含建筑、地形、所有玩家数据、箱子物品等 | ✅ 必选 |
| **玩家数据** | `world/playerdata/` | 玩家的背包、装备、经验、位置（在 world 目录内） | ✅ 必选 |
| **玩家统计** | `world/stats/` | 成就、击杀数等（在 world 目录内） | ✅ 必选 |
| **玩家 advancement** | `world/advancements/` | 进度（在 world 目录内） | ✅ 必选 |
| **区域保护 / 领地数据** | 取决于 Mod（FTB Chunks 等） | 通常在 world 或单独目录 | ⚠️ 视情况 |
| DDNS-GO 配置 | `ddns-go-data/.ddns_go_config.yaml` | 含阿里云 API 密钥 | ⚠️ 必选 |
| 凭据 | `credentials.md` | 所有密码和密钥 | ⚠️ 必选 |
| Mod 配置文件 | `config/` | 各 Mod 的配置设置 | 🔶 建议备份 |
| 服务端配置 | `server.properties` | 游戏规则、难度、视距等 | 🔶 建议备份 |
| KubeJS 脚本 | `kubejs/` | 自定义合成表、脚本等 | 🔶 建议备份 |
| 管理员列表 | `ops.json` | 操作员（OP）列表 | 🔶 建议备份 |
| 白名单 | `whitelist.json` | 白名单玩家 | 🔶 建议备份 |
| 封禁列表 | `banned-players.json`, `banned-ips.json` | 被封禁的玩家和 IP | 🔶 建议备份 |
| 用户名缓存 | `usercache.json`, `usernamecache.json` | 用户名 ↔ UUID 映射 | 📎 可选 |
| TACZ 配置 | `tacz/` | TACZ 枪包自定义数据 | 📎 可选 |
| TLM 资源包 | `tlm_custom_pack/` | 车万女仆资源包 | 📎 可选 |

---

## 迁移步骤

### 新机器上

```bash
# 1. 克隆项目
git clone git@github.com:userzbb/mc-server-1.20.1-forge.git
cd mc-server-1.20.1-forge

# 2. 复制 Mod 文件
cp /path/to/backup/mods/*.jar mods/

# 3. 恢复凭据
cp /path/to/backup/credentials.md .

# 4. 启动 MCSManager
docker compose up -d

# 5. 恢复世界存档（如果备份了）
cp -r /path/to/backup/world mcsm/daemon/data/InstanceData/<uuid>/world

# 6. 在面板中创建实例
#    - 打开 http://服务器IP:23333
#    - 终端 → 主服务器 → 创建实例
#    - 选 Minecraft Java 版 / Forge 1.20.1
#    - 创建后记录实例 UUID（在实例设置页可以看到）

# 7. 配置 Docker 设置
#    编辑 mcsm/daemon/data/InstanceConfig/<新UUID>.json，修改以下字段：
#    - "cwd": "data/InstanceData"
#    - "ports": ["25565:25565/tcp", "25575:25575/tcp", "24454:24454/udp"]
#    - "env": ["EULA=TRUE", "TYPE=FORGE", ...]
#    - "extraVolumes": ["/home/<用户名>/minecraft-server/mods|/mods|ro"]
#      ⚠️ 上面的宿主机路径的 <用户名> 要改成新机器上的用户名
#    详细配置参考 instance-config.json 和 docs/实例配置说明.md

# 7b. 如果新机器上要装 CurseForge 整合包（不使用公共 mods）
#     在 env 中添加 CF_SERVER_MOD，extraVolumes 留空：
#     "env": ["EULA=TRUE", "TYPE=FORGE", "VERSION=1.20.1",
#             "FORGE_VERSION=47.4.21",
#             "CF_SERVER_MOD=https://www.curseforge.com/minecraft/modpacks/<整合包名>",
#             ...]
#     "extraVolumes": []
#     ⚠️ 首次启动时数据目录必须完全为空，否则镜像可能跳过整合包下载。
#     如果启动后发现 mods 目录为空、整合包没下载，按以下步骤修复：
#       a. 面板上停止实例
#       b. sudo docker rm -f <容器名>
#       c. 删除 mcsm/daemon/data/InstanceData/<UUID>/（注意会丢存档）
#       d. 面板上重新开启实例

# 8. 复制配置文件
#    cp server.properties.template mcsm/daemon/data/InstanceData/<新UUID>/server.properties
#    echo "eula=true" > mcsm/daemon/data/InstanceData/<新UUID>/eula.txt

# 9. 修复文件权限（只有用 sudo 创建文件时才需要）
#    容器进程以 uid=1000 运行，如果文件是 root 或其他用户创建的，
#    需要改为容器能写入的 UID。例如：
#    sudo chown 1000:1000 mcsm/daemon/data/InstanceData/<新UUID>/server.properties
#    sudo chown 1000:1000 mcsm/daemon/data/InstanceData/<新UUID>/eula.txt
#    如果文件是当前普通用户创建的（未使用 sudo），则不需要改。

# 10. 重启 daemon，然后在面板启动实例
#     docker compose restart mcsm-daemon
```

> ⚠️ Mod 管理注意事项：`itzg/minecraft-server` 镜像启动时会从 `/mods` 复制文件到实例的 `mods/` 目录，但不会同步删除。要彻底移除一个 Mod，需要同时删除 `mods/` 源文件和实例数据目录里的文件。实例 Mod 数据目录：`mcsm/daemon/data/InstanceData/<UUID>/mods/`

### 旧机器上备份

推荐使用 `backup.sh` 脚本一键备份：

```bash
# 备份整个服务器（世界 + 配置 + 凭据）
./backup.sh

# 备份文件在 backups/ 目录
ls -lh backups/
```

或手动备份：

```bash
# Mod
cp mods/*.jar /backup/

# 世界存档
cp mcsm/daemon/data/InstanceData/<uuid>/world /backup/

# DDNS 配置
cp ddns-go-data/.ddns_go_config.yaml /backup/

# 凭据
cp credentials.md /backup/
```
