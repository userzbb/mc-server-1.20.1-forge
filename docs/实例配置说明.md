# 实例配置说明

> 实例配置模板见 `instance-config.json`。
> 实际运行时配置存储在 `mcsm/daemon/data/InstanceConfig/<UUID>.json`。

---

## Docker 配置

### JSON 格式说明

MCSManager daemon 要求的配置格式是**字符串数组**，不是对象数组。

### 端口映射

```json
"ports": [
    "25565:25565/tcp",
    "25575:25575/tcp",
    "24454:24454/udp"
]
```

格式：`"宿主机端口:容器端口/协议"`

| 宿主机 | 容器 | 协议 | 用途 |
|--------|------|------|------|
| 25565 | 25565 | tcp | Minecraft 游戏 |
| 25575 | 25575 | tcp | RCON 管理 |
| 24454 | 24454 | udp | 语音聊天 |

### 环境变量

```json
"env": [
    "EULA=TRUE",
    "TYPE=FORGE",
    "VERSION=1.20.1",
    "FORGE_VERSION=47.4.21",
    "ONLINE_MODE=false",
    "MEMORY=4G",
    "MAX_PLAYERS=20",
    "TZ=Asia/Shanghai"
]
```

格式：`"KEY=VALUE"`

| 变量 | 值 | 说明 |
|------|-----|------|
| `EULA` | `TRUE` | 同意最终用户许可协议 |
| `TYPE` | `FORGE` | 服务端类型 |
| `VERSION` | `1.20.1` | Minecraft 版本 |
| `FORGE_VERSION` | `47.4.21` | Forge 版本 |
| `ONLINE_MODE` | `false` | 离线模式（允许非正版） |
| `MEMORY` | `4G` | 分配内存 |
| `MAX_PLAYERS` | `20` | 最大玩家数 |
| `TZ` | `Asia/Shanghai` | 时区 |

### 挂载卷

```json
"extraVolumes": [
    "/home/yuan/minecraft-server/mods|/mods|ro"
]
```

格式：`"宿主机路径|容器路径|模式"`（用 `|` 分隔，不是对象）

| 宿主机路径 | 容器路径 | 模式 |
|-----------|---------|------|
| `/home/yuan/minecraft-server/mods` | `/mods` | ro（只读） |

`itzg/minecraft-server` 镜像启动时自动把 `/mods` 下的 `.jar` 复制到服务端的 `mods/` 目录。

### cwd（工作目录）

```json
"cwd": "data/InstanceData/<实例UUID>"
```

`cwd` **不能为空或 null**，daemon 会用它定位实例数据目录。设成相对路径 `data/InstanceData/<UUID>`，daemon 会自动解析为完整的宿主机路径。

### 其他设置

| 字段 | 值 |
|------|-----|
| 网络模式 | `bridge` |
| 内存限制 | 0（不限制） |

---

## 实例数据目录

实例创建后，数据存储在：

```
/home/yuan/minecraft-server/mcsm/daemon/data/InstanceData/<UUID>/
```

包含：
- `world/` — 世界存档
- `mods/` — Mod 文件（由镜像从 `/mods` 自动复制）
- `server.properties` — 服务端配置
- `eula.txt` — EULA 协议

---

## 迁移

把 `instance-config.json` 复制到新服务器的 `mcsm/daemon/data/InstanceConfig/<新UUID>.json` 即可复用这套 Docker 配置。
