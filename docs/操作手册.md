## 三、网络架构

所有服务通过域名 `catzizimiku.top` 走 **IPv6** 访问。不需路由器端口转发。

| 地址 | 用途 |
|------|------|
| `http://catzizimiku.top:23333` | MCSManager 面板（⚠️ HTTP 不是 HTTPS） |
| `catzizimiku.top:25565` | Minecraft 游戏连接 |
| `http://catzizimiku.top:9876` | DDNS-GO 管理界面 |

### MCSManager WebSocket 连接原理

```
浏览器 ──→ catzizimiku.top:24444 ──→ IPv6 直连 daemon
后端容器 ──→ catzizimiku.top:24444 ──→ host-gateway → 宿主机 → daemon
```

`docker-compose.yml` 中 `mcsm-web` 的配置：
```yaml
extra_hosts:
  - "catzizimiku.top:host-gateway"
```

> 节点 IP 必须用域名 `catzizimiku.top`。不能用 Docker 容器名（浏览器不认识），不能用内网 IP（走不了 IPv6）。

### 创建 Minecraft 实例（需人工操作）

MCSManager 的实例创建必须通过 Web 面板 UI 操作，API 不支持直接创建。

操作步骤：
1. 打开 `http://catzizimiku.top:23333`
2. 登录后点左侧 **「终端」** → 点击 **「主服务器」** 节点
3. 点 **「创建实例」**
4. 类型选 **Minecraft Java 版**，版本选 **Forge 1.20.1**
5. 填写实例名称，确认创建
6. 创建后进入实例 → **「文件管理」**
7. 上传 Mod（`mods/` 目录下的 `.jar`）和世界存档
8. 回到控制台 → 点 **「开启实例」**

### AI 可以做的事

实例创建后，AI 可以帮用户上传 Mod 文件、配置 Docker 设置、修复权限。

实例数据存储在 `mcsm/daemon/data/InstanceData/<实例UUID>/` 目录下。
Mod 源文件在 `mods/` 目录。

```bash
# 查找实例 UUID
ls /home/yuan/minecraft-server/mcsm/daemon/data/InstanceConfig/
cat /home/yuan/minecraft-server/mcsm/daemon/data/InstanceConfig/<UUID>.json | python3 -c "import sys,json; print(json.load(sys.stdin)['nickname'])"

# 复制 Mod 到实例数据目录
cp /home/yuan/minecraft-server/mods/*.jar /home/yuan/minecraft-server/mcsm/daemon/data/InstanceData/<UUID>/mods/

# 复制世界存档
cp -r /home/yuan/minecraft-server/mcsm/daemon/data/InstanceData/<UUID>/world /home/yuan/minecraft-server/mcsm/daemon/data/InstanceData/<UUID>/
```

### Docker 实例配置格式

当通过 `instance-config.json` 模板或 `mcsm/daemon/data/InstanceConfig/<UUID>.json` 配置实例时，必须使用正确的格式：

| 字段 | 格式 | 示例 |
|------|------|------|
| `cwd` | 非空相对路径 | `"data/InstanceData"` |
| `ports` | 字符串数组 | `["25565:25565/tcp"]` |
| `env` | `KEY=VALUE` 字符串数组 | `["EULA=TRUE"]` |
| `extraVolumes` | `源路径\|目标路径\|模式` 字符串数组 | `["/home/user/mods\|/mods\|ro"]` |

**注意：**
- `cwd` 不能为空或 null，daemon 会报错
- 端口、环境变量、挂载卷都是**字符串格式**，不是对象
- 挂载卷用 `|` 分隔，不是冒号

### 修复文件权限

实例创建后，`server.properties` 和 `eula.txt` 需要改为容器用户可写：

```bash
INST_DIR="/home/yuan/minecraft-server/mcsm/daemon/data/InstanceData/<UUID>"
sudo chown 1000:1000 ${INST_DIR}/server.properties ${INST_DIR}/eula.txt
```

### 守护进程节点配置

存储在 `mcsm/web/data/RemoteServiceConfig/<UUID>.json`：
```json
{
    "ip": "catzizimiku.top",
    "port": 24444,
    "apiKey": "见 credentials.md",
    "remarks": "主服务器"
}
```

### 排查面板连接问题

```bash
# 1. 确认 daemon 运行
docker compose ps mcsm-daemon

# 2. 从 Web 容器测试连通性
docker exec mcsm-web node -e "const http = require('http'); http.get('http://catzizimiku.top:24444/', r => { r.resume(); console.log(r.statusCode) }).on('error', e => console.log(e.message))"

# 3. 预期返回 200
docker compose exec mcsm-web cat /etc/hosts | grep catzizimiku

# 4. 重启面板
docker compose restart mcsm-web
```

## 八、Mod 管理

### 目录

```
/home/yuan/minecraft-server/mods/  ← 放 .jar 文件（源目录）
```

上传到 MCSManager 实例时，从 `mods/` 复制到实例的 `InstanceData/<UUID>/mods/` 目录。

### 更新 Mod

**添加：** 下载 `.jar` 放入 `mods/` → 重启实例（容器自动复制到实例）

> ⚠️ 创建新实例时，不要默认挂载 `mods/` 目录。必须等用户确认需要哪些 Mod 后再添加。

**删除：**
1. 从 `mods/` 目录删除对应的 `.jar`
2. 还需要删除实例数据目录里对应的文件，否则重启后旧的还在：
   ```bash
   rm /home/yuan/minecraft-server/mcsm/daemon/data/InstanceData/<UUID>/mods/<文件名>
   ```
3. 重启实例生效

### 禁止安装的 Mod（会导致服务器崩溃）

| Mod | 原因 |
|-----|------|
| `entity_texture_features` | 客户端 Mixin → 服务器崩溃 |
| `oculus` | 客户端渲染 Mixin → 崩溃 |
| `sodiumextras` | 客户端渲染 Mixin → 崩溃 |
| `DGLabCraft` | 客户端 UI Mixin → 崩溃 |
