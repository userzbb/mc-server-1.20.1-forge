## 二、使用流程

```
1. docker compose up -d           → 部署 MCSManager + DDNS-GO
2. 浏览器访问面板 http://...:23333 → 创建 Minecraft 服务器实例
3. 在面板里启动/停止/管理服务器    → 开玩
```

## 三、网络架构

所有服务通过域名 `catzizimiku.top` 走 **IPv6** 访问。不需路由器端口转发。

| 地址 | 用途 |
|------|------|
| `http://catzizimiku.top:23333` | MCSManager 面板（⚠️ HTTP 不是 HTTPS） |
| `catzizimiku.top:25565` | Minecraft 游戏连接 |
| `http://catzizimiku.top:9876` | DDNS-GO 管理界面 |

### MCSManager WebSocket 连接原理

```
浏览器 ──→ catzizimiku.top:24444 ──→ IPv6 直连 daemon
后端容器 ──→ catzizimiku.top:24444 ──→ host-gateway → 宿主机 → daemon
```

`docker-compose.yml` 中 `mcsm-web` 的配置：
```yaml
extra_hosts:
  - "catzizimiku.top:host-gateway"
```

> 节点 IP 必须用域名 `catzizimiku.top`。不能用 Docker 容器名（浏览器不认识），不能用内网 IP（走不了 IPv6）。

## 四、日常操作命令

```bash
cd /home/yuan/minecraft-server

# 启动所有服务
docker compose up -d

# 停止（不删除容器）
docker compose stop

# 重启
docker compose restart mcsm-web

# 查看状态
docker compose ps

# 完全停止并清理
docker compose down

# 日志
docker compose logs -f mcsm-web
docker compose logs -f mcsm-daemon
docker compose logs -f ddns-go
```

## 五、MCSManager 管理

### 凭据

所有密码和密钥在 `credentials.md`：
- MCSManager 面板账号密码
- 守护进程密钥
- RCON 密码
- DDNS-GO 阿里云 API 密钥
- sudo 密码

### 创建 Minecraft 实例

1. 面板 → 左侧「终端」→ 点击「主服务器」节点
2. 点击「创建实例」
3. 类型选 **Minecraft Java 版**，版本选 **Forge 1.20.1**
4. 创建后进入实例 →「文件管理」
5. 上传 Mod（`mods/` 目录下的 `.jar`）和世界存档
6. 回到控制台 → 点击「开启实例」

> 如果面板里看不到实例数量，刷新页面(Ctrl+F5)或重新登录。

### 守护进程节点配置

存储在 `mcsm/web/data/RemoteServiceConfig/<UUID>.json`：
```json
{
    "ip": "catzizimiku.top",
    "port": 24444,
    "apiKey": "见 credentials.md",
    "remarks": "主服务器"
}
```

### 排查面板连接问题

```bash
# 1. 确认 daemon 运行
docker compose ps mcsm-daemon

# 2. 从 Web 容器测试连通性
docker exec mcsm-web node -e "const http = require('http'); http.get('http://catzizimiku.top:24444/', r => { r.resume(); console.log(r.statusCode) }).on('error', e => console.log(e.message))"

# 3. 预期返回 200
docker compose exec mcsm-web cat /etc/hosts | grep catzizimiku

# 4. 重启面板
docker compose restart mcsm-web
```

## 六、防火墙

### 确认规则

```bash
sudo firewall-cmd --info-service=mc-server
sudo firewall-cmd --zone=FedoraWorkstation --list-services | grep mc-server
sudo firewall-cmd --zone=FedoraServer --list-services | grep mc-server
```

> 物理网卡 `enp2s0` 绑定 `FedoraServer` 区域。两个区域都必须启用 `mc-server` 服务。

### 端口清单

| 端口 | 协议 | 用途 |
|------|------|------|
| 25565 | TCP+UDP | Minecraft 游戏 |
| 23333 | TCP | MCSManager 面板 |
| 24444 | TCP | MCSManager 守护进程 |
| 25575 | TCP+UDP | RCON |
| 24454 | TCP+UDP | 语音聊天 |
| 9876 | TCP+UDP | DDNS-GO |

## 七、DDNS-GO

```
管理地址: http://catzizimiku.top:9876
配置文件: ddns-go-data/.ddns_go_config.yaml（root 权限）
DNS 服务商: 阿里云 DNS
域名: catzizimiku.top
支持: IPv4 + IPv6 双栈
```

```bash
host catzizimiku.top                                 # 检查 DNS
curl -4 https://myip.ipip.net; curl -6 https://myip.ipip.net  # 公网 IP
docker compose restart ddns-go                        # 重启
docker compose logs ddns-go                           # 日志
```

## 八、Mod 管理

### 目录

```
/home/yuan/minecraft-server/mods/  ← 放 .jar 文件
```

### 更新 Mod

1. 下载 `.jar` 放入 `mods/` 目录
2. 在 MCSManager 面板中重新创建实例（或重启已有实例）

### 禁止安装的 Mod（会导致服务器崩溃）

| Mod | 原因 |
|-----|------|
| `entity_texture_features` | 客户端 Mixin → 服务器崩溃 |
| `oculus` | 客户端渲染 Mixin → 崩溃 |
| `sodiumextras` | 客户端渲染 Mixin → 崩溃 |
| `DGLabCraft` | 客户端 UI Mixin → 崩溃 |
