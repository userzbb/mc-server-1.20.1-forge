## 二、网络架构

### 访问方式

所有服务通过域名 `catzizimiku.top` 走 **IPv6** 访问。不需路由器端口转发。

| 地址 | 用途 |
|------|------|
| `http://catzizimiku.top:23333` | MCSManager 面板（⚠️ HTTP 不是 HTTPS） |
| `catzizimiku.top:25565` | Minecraft 游戏连接 |
| `http://catzizimiku.top:9876` | DDNS-GO 管理界面 |

### MCSManager WebSocket 连接原理

```
浏览器 ──→ catzizimiku.top:24444 ──→ IPv6 直连 daemon
后端容器 ──→ catzizimiku.top:24444 ──→ host-gateway → 宿主机 → daemon
```

`docker-compose.yml` 中 `mcsm-web` 的配置：
```yaml
extra_hosts:
  - "catzizimiku.top:host-gateway"  # 后端通过宿主机转发连 daemon
```

> 节点 IP 必须用域名 `catzizimiku.top`。不能用 Docker 容器名（浏览器不认识），不能用内网 IP（走不了 IPv6）。

## 三、日常操作命令

### 服务生命周期

```bash
cd /home/yuan/minecraft-server

# 启动所有服务
docker compose up -d

# 停止（不删除容器）
docker compose stop

# 重启单个服务
docker compose restart mc-forge
docker compose restart mcsm-web

# 查看状态
docker compose ps

# 完全停止并清理
docker compose down
```

### 日志

```bash
docker compose logs -f mc-forge        # MC 服务器
docker compose logs -f mcsm-web         # MCSManager 面板
docker compose logs -f mcsm-daemon      # 守护进程
docker compose logs -f ddns-go          # DDNS
```

## 四、MCSManager 配置

### 凭据

所有密码和密钥统一记录在 `credentials.md`（已 gitignore），包括：

- MCSManager 面板账号密码
- 守护进程密钥
- RCON 密码
- DDNS-GO 阿里云 API 密钥
- sudo 密码

### 守护进程节点配置

存储在 `mcsm/web/data/RemoteServiceConfig/<UUID>.json`，内容示例：
```json
{
    "ip": "catzizimiku.top",
    "port": 24444,
    "apiKey": "见 credentials.md",
    "remarks": "主服务器"
}
```

### 排查面板连接问题

```bash
# 1. 确认 daemon 运行
docker compose ps mcsm-daemon

# 2. 从 Web 容器测试 daemon 连通性
docker exec mcsm-web node -e "const http = require('http'); http.get('http://catzizimiku.top:24444/', r => { r.resume(); console.log(r.statusCode) }).on('error', e => console.log(e.message))"

# 3. 预期返回 200。如果不是，检查 extra_hosts 配置
docker compose exec mcsm-web cat /etc/hosts | grep catzizimiku

# 4. 重启面板
docker compose restart mcsm-web
```

## 五、防火墙

### 确认规则

```bash
# 查看 mc-server 服务包含的端口
sudo firewall-cmd --info-service=mc-server

# 查看哪些区域启用了 mc-server
sudo firewall-cmd --zone=FedoraWorkstation --list-services | grep mc-server
sudo firewall-cmd --zone=FedoraServer --list-services | grep mc-server
```

> 物理网卡 `enp2s0` 绑定 `FedoraServer` 区域。两个区域都必须启用 `mc-server` 服务。

### mc-server 服务端口清单

| 端口 | 协议 | 用途 |
|------|------|------|
| 9876 | TCP+UDP | DDNS-GO |
| 25565 | TCP+UDP | Minecraft 游戏 |
| 25575 | TCP+UDP | RCON |
| 24454 | TCP+UDP | 语音聊天 |
| 23333 | TCP | MCSManager 面板 |
| 24444 | TCP | MCSManager 守护进程 |

### 命令行操作

```bash
# 放行端口
sudo firewall-cmd --add-port=23333/tcp --permanent

# 重载
sudo firewall-cmd --reload
```

## 七、Mod 管理

### 目录

```
/home/yuan/minecraft-server/mods/  ← 放 .jar 文件
```

### 更新步骤

1. 下载 `.jar` 放入 `mods/` 目录
2. `docker compose restart mc-forge`

### 禁止安装的 Mod（会导致服务器崩溃）

| Mod | 原因 |
|-----|------|
| `entity_texture_features` | 客户端 Mixin → 服务器崩溃 |
| `oculus` | 客户端渲染 Mixin → 崩溃 |
| `sodiumextras` | 客户端渲染 Mixin → 崩溃 |
| `DGLabCraft` | 客户端 UI Mixin → 崩溃 |

> 其他纯客户端 mod（embeddium, JEI 等）不会崩溃，但放服务器上也没意义。
