## 2. 日常操作

### 启动 / 停止 / 重启 所有服务

```bash
cd ~/minecraft-server

# 启动（后台）
docker compose up -d

# 停止
docker compose stop

# 重启
docker compose restart

# 查看运行状态
docker compose ps

# 完全停止并删除容器
docker compose down
```

### 查看日志

```bash
# Minecraft 服务器日志
docker compose logs -f mc-forge

# MCSManager 日志
docker compose logs -f mcsm-web
docker compose logs -f mcsm-daemon

# DDNS-GO 日志
docker compose logs -f ddns-go
```

### 更新 Mod

1. 下载 `.jar` 文件放到 `~/minecraft-server/mods/` 目录
2. 重启服务器：`docker compose restart mc-forge`

## 4. 防火墙

所有端口统一集成在 `mc-server` 这个 firewalld 服务里。

### 在 GUI 里修改

终端运行：`! firewall-config`
在打开的窗口里：
- **服务 → mc-server**：可以编辑这个服务包含的端口
- **区域 → FedoraWorkstation**：勾选 `mc-server` 服务

### 命令行查看

```bash
# 查看 mc-server 服务包含的端口
sudo firewall-cmd --info-service=mc-server

# 查看启用了哪些服务
sudo firewall-cmd --list-services

# 重载防火墙
sudo firewall-cmd --reload
```

## 5. DDNS-GO 动态域名

### 访问

```
http://catzizimiku.top:9876
```

### 功能

把你的公网 IP 自动绑定到域名 `catzizimiku.top`，IP 变了域名自动更新。

支持：
- ✅ IPv4 + IPv6 双栈
- ✅ 阿里云 DNS

### 如果域名解析不对

```bash
# 检查 DNS 解析
host catzizimiku.top

# 检查当前公网 IP
curl -4 https://myip.ipip.net
curl -6 https://myip.ipip.net

# 重启 DDNS-GO
docker compose restart ddns-go

# 查看 DDNS-GO 日志
docker compose logs ddns-go
```

## 6. Mod 管理

### Mod 清单

完整列表见 `mod-list.md`。

### Mod 目录

```
~/minecraft-server/mods/    ← 放 jar 文件的地方（只读挂载给容器）
```

### ⚠️ 注意

**不要安装这些客户端 Mod 到服务器上**，会崩溃：

| Mod | 原因 |
|-----|------|
| `entity_texture_features` | Mixin 引用客户端类 → ❌ 崩溃 |
| `oculus` | 客户端光影 Mixin → ❌ 崩溃 |
| `sodiumextras` | 客户端渲染 Mixin → ❌ 崩溃 |
| `DGLabCraft` | 客户端 UI Mixin → ❌ 崩溃 |

其他客户端 mod（如 embeddium、JEI 等）虽然不会崩，但也没必要放服务器上。

## 7. 常见问题

### Q: 公网连不上怎么办？

**IPv4 方式：**
服务器在路由器后面（`192.168.1.11`），需要路由器做端口转发：
1. 浏览器打开 `http://192.168.1.1`
2. 找到「端口转发」或「虚拟服务器」
3. 把需要的端口转发到 `192.168.1.11`

**IPv6 方式（推荐）：**
如果你的网络有 IPv6，直接连接即可。域名 `catzizimiku.top` 已绑定了 IPv6。

### Q: Docker 拉取镜像太慢/超时？

配置了国内镜像加速 `docker.1ms.run`，如果还慢可以换一个：

```bash
sudo tee /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": ["https://docker.1ms.run"]
}
EOF
sudo systemctl restart docker
```

其他可用镜像源：
- `https://docker.m.daocloud.io`
- `https://dockerproxy.com`

### Q: 服务起不来，端口被占用？

```bash
# 查谁占了端口
sudo ss -tlnp | grep <端口号>

# 最常见的冲突：mc-forge 和 MCSManager 创建的实例用了同个端口
# 解决方法：关掉一个
```

### Q: 想重置整个服务器？

```bash
cd ~/minecraft-server
docker compose down -v          # 停止并删掉所有容器和网络
rm -rf data/ mcsm/               # 清除数据
git checkout -- .                # 恢复配置文件到 git 版本
docker compose up -d             # 重新启动
```
